    Introduction

    Welcome to the Todo list API! You can use our API to access Todo list API endpoints, which can get information on user projects and tasks.

    Authentication

    To use API, you need username and password. Using this data you can get token.

    To authenticate, use this code:

    POST http://127.0.0.1:8080/auth
Content-Type: application/json
Cache-Control: no-cache

{ "username": "email@test.com", "password": "secretpassword"}

    Response:

    Code: 200

    {
  "token": "xmc8kPNapdfbrEAUZ15VkSE8qQudJAVg",
  "expires_at": "2019-08-27 20:57:53"
}

    POST /auth

    This endpoint retrieves user token.

    Parameters

    Parameter Type Description Required
    username string Unique username +
    password string Password +
    You must use received token in request headers.

    Errors

    The Todo list API uses the following error codes:

    Error Code Description
    400 Bad Request - Your request is invalid.
    401 Unauthorized - Your API key is wrong.
    403 Forbidden - The requested for administrators only.
    404 Not Found - The resource could not be found.
    405 Method Not Allowed - You tried to access resource with an invalid method.
    406 Not Acceptable - You requested a format that isn't json.
    500 Internal Server Error - We had a problem with our server. Try again later.
    503 Service Unavailable - We're temporarily offline for maintenance. Please try again later.
